## Cross-cutting rules
- **T3-CC-1 — held-out is mechanically separated + never tuned.** `Panel` keeps `dev_teams` and
  `heldout_teams` as **separate** lists. Dev schedule generation uses ONLY dev teams; held-out generation
  requires an explicit `confirm_heldout=True` and is refused otherwise. A test proves a dev schedule
  never contains a held-out team. Discipline (inherits parent CC-5): **no hyperparameter is ever read
  from held-out results** — only at the final gate (T6).
- **T3-CC-2 — every team is legal + hashed.** Each team `.txt` passes `validate-team gen9vgc2025regi`
  (exit 0) and has a `pack-team`-generated `.packed`; `panel_hash` covers the team identities so a team
  swap changes the hash.
- **T3-CC-3 — reproducible policies are deterministic.** Paired comparison (T1b) needs deterministic
  bots. `heuristic`, `greedy_protect`, `simple_heuristic`, `scripted_vgc` are deterministic; `max_damage`
  is made deterministic (T3c fixes its `pick_random_pair` fallback). `random` stays intentionally
  non-deterministic and is flagged **non-reproducible** wherever used (stress policy, not for paired Δ).
- **T3-CC-4 — provenance travels.** `panel_hash` (+ the T1c `schedule_hash`) is recorded on every
  generated schedule and carried into T2 rows' context so T5 can group by panel.

@dataclass(frozen=True)
class PanelTeam:

    heldout_teams: tuple[PanelTeam, ...]
    panel_hash: str
